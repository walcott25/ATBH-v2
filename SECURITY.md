# Security Policy

## Reporting a Vulnerability

If you discover a security vulnerability, please do **not** open a public issue.

Instead, send a detailed report to **asuogyamantourismboard@gmail.com** with the subject **"Security Vulnerability: [brief description]"**.

Please include:
- A description of the vulnerability
- Steps to reproduce
- Potential impact
- Any suggested mitigation

We will acknowledge receipt within 48 hours and work on a fix before public disclosure.

## Scope

This policy covers:
- The live application at https://atbh.vercel.app
- All code in this repository
- API endpoints and serverless functions

Out of scope: third-party services (Clerk, Paystack, Vercel, NVIDIA) — report those to their respective security teams.

## Responsible Disclosure

We ask that you give us reasonable time to fix the issue before disclosing it publicly. We will coordinate disclosure with you once a fix is deployed.
